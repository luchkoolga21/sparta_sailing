# Правила проекта Yacht Rental Design

## Общие принципы

### 1. Коммуникация и код
- **Диалог**: Весь диалог с разработчиком ведется на **русском языке**.
- **Код**: Все переменные, функции, классы, комментарии в коде пишутся на **английском языке**.

### 2. Технологический стек
- **HTML5**: Семантическая разметка, без фреймворков.
- **CSS3**: Vanilla CSS, без препроцессоров и CSS-in-JS.
- **JavaScript (ES6+)**: Современный JavaScript, без тяжелых фреймворков (React, Vue, Angular).
- **Принцип**: Простота превыше всего. Если нужна библиотека или фреймворк — необходима веская аргументация.

### 3. Структура и организация кода
- **Простота**: Код должен быть простым и структурированным.
- **Без излишеств**: Никаких лишних блоков, файлов или библиотек.
- **Модульность**: Каждый раздел сайта (блок) должен быть независимым модулем.
- **Изолированность**: Удаление или изменение одного модуля не должно ломать другие модули или общую структуру сайта.

### 4. Открытые источники
- **Иконки**: Используем открытые библиотеки иконок (например, Iconify).
- **Шрифты**: Используем Google Fonts или другие open-source шрифты.
- **Лицензирование**: Все используемые ресурсы должны иметь открытую лицензию.

### 5. Адаптивность
- **Мультиразмерный формат**: Сайт должен корректно отображаться на всех устройствах.
- **Десктоп**: Оптимизация для больших экранов (1920px+).
- **Планшет**: Адаптация для средних экранов (768px - 1024px).
- **Мобильные устройства**: Полная адаптация для смартфонов (320px - 767px).
- **Подход**: Mobile-first или desktop-down — в зависимости от задачи.

## Архитектурные правила

### 1. Модульная структура
```
project/
├── index.html           # Главный HTML-файл
├── style.css            # Единый файл стилей (с модульной структурой через комментарии)
├── script.js            # Единый файл скриптов (с модульной структурой)
├── assets/              # Ресурсы (если понадобятся)
│   ├── images/
│   └── icons/
├── .agent/              # Рабочие файлы AI-агента
│   └── workflow/
└── PROJECT_RULES.md     # Этот файл
```

### 2. CSS: Модульность через комментарии
```css
/* ==================== MODULE_NAME ==================== */
/* Стили модуля */

/* ==================== ANOTHER_MODULE ==================== */
/* Стили другого модуля */
```

### 3. JavaScript: Модульность через функции
```javascript
// ==================== MODULE_NAME ====================
function initModuleName() {
    // Логика модуля
}

// ==================== ANOTHER_MODULE ====================
function initAnotherModule() {
    // Логика другого модуля
}
```

### 4. HTML: Семантическая структура
```html
<header>...</header>
<main>
    <section id="module-1">...</section>
    <section id="module-2">...</section>
</main>
<footer>...</footer>
```

## Правила работы с дизайном

### 1. Референсы
- Дизайн берется с референсного сайта (например, brand.ai).
- Все токены (цвета, шрифты, отступы) извлекаются и документируются.
- Реализация максимально точная (pixel-perfect).

### 2. CSS Variables
- Все дизайн-токены определяются в `:root`.
- Использование переменных обязательно для консистентности.

```css
:root {
    --color-primary: #1B1B1B;
    --font-heading: 'Inter', sans-serif;
    --space-md: 24px;
}
```

### 3. Производительность
- Минимум JavaScript.
- Оптимизация изображений.
- Lazy loading для изображений (если их много).

## Правила разработки

### 1. Тестирование
- Проверка на всех разрешениях (320px, 768px, 1024px, 1920px).
- Проверка в разных браузерах (Chrome, Safari, Firefox).
- Проверка доступности (accessibility).

### 2. Комментарии
- **CSS**: Четкие разделители между модулями.
- **JavaScript**: Комментарии на английском, описывающие логику.
- **HTML**: Минимум комментариев, только для сложных структур.

### 3. Версионирование
- Использование Git для контроля версий.
- Коммиты на английском языке.
- Четкие описания изменений в коммитах.

## Запреты

❌ **Нельзя использовать**:
- Фреймворки (React, Vue, Angular) без **веской аргументации**.
- Препроцессоры CSS (SASS, LESS) — только vanilla CSS.
- jQuery или другие устаревшие библиотеки.
- Inline-стили в HTML (за исключением критических случаев).
- Неоптимизированные изображения большого размера.
- Сторонние библиотеки без open-source лицензии.
