# Role
You are an expert Senior Frontend Developer and UI/UX Designer specialized in high-conversion luxury tourism websites. Your task is to implement a Single Page Application (Landing Page) for a private yacht rental service in Nakhodka (Russia).

# Context
The website sells **experiences and emotions**, not just a boat rental. The focus is on the beauty of the Primorye nature (cliffs, islands, fog, sunsets). The yacht is secondary—it's just a comfortable tool to see this beauty.

# Technical Stack
*   **Core:** HTML5, CSS3, Vanilla JavaScript (ES6+).
*   **No Frameworks:** Keep it lightweight and fast.
*   **Styling:** CSS Variables for theming, Flexbox/Grid for layout.
*   **Responsiveness:** Mobile-First approach (critical, as most traffic is mobile).

# Design Requirements (The "Vibe")
1.  **Dynamic Atmosphere (Unique Feature):**
    *   Implement a "Day/Sunset" toggle switch.
    *   **Day Mode:** Bright, turquoise water, white text, fresh feel.
    *   **Sunset/Night Mode:** Warm orange/purple tones, dark backgrounds, cozy lighting.
    *   *Note:* Transitions must be smooth (transition: all 0.8s ease).
2.  **Visual Focus:**
    *   **Nature First:** Use placeholders depicting cliffs, misty mornings, calm bays, and sunsets.
    *   **Fog Aesthetic:** Include a "Mystic Primorye" feel in the gallery or backgrounds.
    *   **POV:** Images should look like they are taken from the guest's eyes (legs on the deck, hand with a glass, view of the horizon).

# Specific Implementation Rules

## 1. Semantic HTML5
Strictly separate the layout into three main areas:
- `<header>`: For the navigation/top bar.
- `<main>`: To wrap the page content. Inside it, place the Hero section using `<section id="hero">` and all other sections.
- `<footer>`: For the bottom area.
- Ensure the `<main>` tag is the only direct child of `<body>` besides `<header>` and `<footer>` (and the theme toggle button).

## 2. Modularity
Ensure every section is independent. Do not nest the Hero section or any other section inside the Header. Sections should follow one after another within the `<main>` tag.

## 3. Code Organization
Add clear comments in both HTML and CSS (e.g., `/* --- Hero Section --- */`) to visually separate these blocks. Maintain a clear hierarchy in CSS using comments to mark the start and end of component styles.

## 4. Layout
Use a common `.container` class to align content centrally across the Header, Hero, and Footer. Always use the `.container` for max-width and centering.

## 5. Development Workflow
- Create `style.css` with CSS Variables for colors (`--bg-color`, `--text-color`, `--accent-color`).
- Implement the JS logic for the Theme Toggle class switching on `<body>`.
- Ensure strictly responsive design for mobile screens (hamburger menu if needed).
